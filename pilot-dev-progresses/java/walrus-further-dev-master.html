Further development of the walrus project

Preliminaries

Walrus (project homepage: http://sourceforge.net/projects/walruscms/) is a CMS application written in Java. While it has many interesting features, it is now considerable outdated. Our further development here shall try to bring the project up-to-date to reflect state-of-art technologies.

Bugfixes

  1. bug-201506-24-1-improved-compatibility-and-dependency

    The latest (1.5.0) software could be improved in terms of compatibility and dependency in many aspects:

    • The latest 1.5.0 walrus app does not work in the login portal url is '/cms/login', but we prefer relative urls than absolute ones such that they are really 'portable'. Also, deployment methods evolved from jetty 7 to 9; we'd like walrus to be compatible with them all as much as possible.

    • Also, walrus is dependent on many modules not found on common maven repositories. This makes it impossible to build without much effort on building customized maven proxy servers, etc. We'd like to remove such dependencies as much as possible.

    • The directory holding the uploaded files shall be defined more reasonably.

  2. bug-201506-24-2-i18n

    The internationalization is incomplete and not clean; we shall fix that.
